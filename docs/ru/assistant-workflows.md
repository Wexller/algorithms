# Как обращаться к ассистенту

## Когда нужна теория до решения

Используй такой запрос:

```text
Объясни модуль sliding-window на примере локальных задач из этого проекта.
Мне нужны: интуиция, инварианты, типовые ошибки и признаки, что задача решается окном.
Не решай за меня текущую задачу полностью.
```

## Когда нужна валидация после решения

Используй такой запрос:

```text
Проверь мое решение задачи problems/<nn-stage>/<nn-module>/<slug>.
Нужен review в формате:
1. bugs
2. complexity risks
3. missing tests
4. correctness argument
```

## Когда нужна помощь в тупике

Используй такой запрос:

```text
Я застрял на problems/<nn-stage>/<nn-module>/<slug>.
Не давай готовый код сразу.
Сначала укажи, какой паттерн здесь вероятнее всего подходит, какие есть инварианты и какой baseline стоит рассмотреть.
```

## Когда нужно добить качество

Используй такой запрос:

```text
Предложи дополнительные тесткейсы для problems/<nn-stage>/<nn-module>/<slug>.
Сделай акцент на граничные случаи и сценарии, где ломаются наивные реализации.
```
