# Учебный трек

## Принцип

Проект устроен как локальная академия алгоритмов: ты идешь по стадиям, решаешь обязательные задачи, фиксируешь ошибки, затем переходишь дальше только после локальной проверки.

## Текущие стадии

### 1. Foundations

- `arrays`
- `strings`
- `hash-maps`

Цель: научиться видеть базовую структуру задачи, считать сложность и быстро отличать перебор от линейных решений.

### 2. Patterns

- `two-pointers`
- `sliding-window`
- `binary-search-intervals`
- `stack-queue`

Цель: освоить повторяющиеся паттерны, которые покрывают большую долю interview-style задач.

### 3. Structures

- `linked-list`
- `trees-bst`
- `heap-backtracking`

Цель: уверенно работать со структурами данных, рекурсией, обходами и поиском.

### 4. Advanced

- `graphs`
- `dynamic-programming`

Цель: дойти до задач, где уже мало просто знать шаблон и нужно уверенно строить состояние, графовую модель и инварианты.

## Следующая волна после текущего каталога

- `greedy`
- `union-find`
- `trie`
- `topological-sort`
- `shortest-paths`
- `monotonic-stack-queue`
- `advanced-dp`
- `segment-tree-fenwick`
- `mock-interviews`

## Правило перехода

- Сначала закрываются все `required: true` задачи модуля.
- Затем допускается переход к следующему модулю.
- Опциональные задачи используются для закрепления и проверки глубины.

## Минимальный стандарт mastery

Для каждого solved-решения ты должен уметь:

- объяснить идею без кода;
- назвать инвариант;
- защитить выбранную сложность;
- перечислить крайние случаи;
- написать решение заново без подглядывания;
- сказать, чем твое решение лучше наивного.
